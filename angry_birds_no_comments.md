
# Angry Birds 网页小游戏 —— 技术实现规格文档

## 1. 项目概述

本项目目标是实现一个可在浏览器中运行的 **Angry Birds（愤怒的小鸟）** 风格的 2D 物理弹射游戏，并通过 **GitHub Pages** 进行托管。玩家通过拖拽弹弓发射小鸟，击倒由不同材质方块搭建的结构、消灭其中的小猪，以通关并获得分数。

**面向对象**：本文档供 AI coding agents 阅读并据此生成完整、可运行的代码。Agent 应在遵循下述约束的前提下补全所有未明确指定的实现细节，并保证最终产物可直接部署。

---

## 2. 技术选型与约束

为了能在 GitHub Pages（纯静态托管，无后端）上运行，必须满足以下约束：

- **纯前端、纯静态**：只能使用 HTML / CSS / JavaScript，不依赖任何服务器端逻辑或数据库。所有资源（脚本、图片、音频）都打包进仓库。
- **物理引擎**：推荐使用 **Matter.js**（轻量、文档完善、适合刚体碰撞与弹射）。渲染层可用 Matter.js 自带的 Render，或用 **HTML5 Canvas** 自绘以获得更好的美术控制。本文档默认 **Matter.js 负责物理 + 自定义 Canvas 渲染**。
- **构建方式优先零构建**：优先使用原生 ES Modules + CDN 引入（如 jsDelivr 上的 Matter.js），这样可以直接把仓库根目录的 `index.html` 部署到 Pages，无需打包步骤。如需打包，使用 Vite，并将 `base` 配置为仓库名以适配 Pages 路径。
- **资源生成**：图片素材优先用 Canvas 程序化绘制或内联 SVG，避免引入版权素材。音效可使用 Web Audio API 程序化合成，或使用明确无版权（CC0）的音频文件。
- **不使用任何付费 / 需鉴权的第三方 API**。

## 4. 核心游戏功能

### 4.1 弹弓发射机制（最核心）

这是游戏体验的灵魂，需要重点打磨：

- 玩家在弹弓上的小鸟处 **按下并拖拽**（鼠标或触摸），松开后小鸟以与拖拽方向相反、与拖拽距离成正比的初速度飞出。
- 拖拽时实时绘制 **预测轨迹线**（用一串半透明圆点表示），帮助玩家瞄准。轨迹通过对初速度做匀加速运动公式逐帧采样计算：\(x(t) = x_0 + v_x t,\quad y(t) = y_0 + v_y t + \tfrac{1}{2} g t^2\)。
- 拖拽距离应有 **最大上限**（钳制 magnitude），超过后小鸟停在最大拉伸半径上，避免速度无限大。
- 松手后施加初速度：\(\vec{v} = k \cdot (\vec{P}_{anchor} - \vec{P}_{release})\)，其中 \(k\) 为发射力度系数，需调参。
- 弹弓两条皮筋的视觉绘制（拖拽时皮筋跟随，松手后回弹动画）。

### 4.2 小鸟种类与技能

至少实现一种，建议实现多种以增加趣味。点击屏幕触发空中技能（每只鸟在飞行中只能用一次）：

- **红鸟（Red）**：无特殊技能，标准弹射。
- **黄鸟（Chuck）**：点击后向前加速冲刺，对木质结构伤害高。
- **蓝鸟（Blues）**：点击后分裂成三只，对冰块伤害高。
- **黑鸟（Bomb）**：点击或落地后爆炸，产生范围冲击力。
- **白鸟（Matilda）**：点击后投下炸蛋并向上反弹。

### 4.3 关卡系统

- 关卡以 **数据驱动**：用 JSON / JS 对象描述每关的方块、小猪、小鸟队列、地形与三星阈值。便于 agent 扩展与玩家自定义。
- 关卡数据结构示例：

```js
{
  id: 1,
  birds: ["red", "yellow", "blue"],     // 发射队列
  blocks: [
    { type: "wood", x: 600, y: 400, w: 20, h: 120, angle: 0 },
    { type: "stone", x: 700, y: 350, w: 120, h: 20 }
  ],
  pigs: [
    { x: 650, y: 360, radius: 20 }
  ],
  stars: { two: 10000, three: 20000 } // 分数阈值
}
```

### 4.4 材质与破坏系统

不同材质有不同密度、强度（HP）和音效：

- **木**：中等强度，黄鸟克制。
- **冰/玻璃**：低强度，易碎，蓝鸟克制。
- **石**：高强度，黑鸟爆炸克制。
- 方块受到的碰撞冲量超过其阈值时进入“受损”状态（换贴图），再次超过则销毁并加分。冲量可从 Matter.js 的 `collisionStart` 事件中读取相对速度估算。

### 4.5 胜负判定

- **胜利**：所有小猪被消灭。
- **失败**：所有小鸟用尽且仍有小猪存活。
- 小猪与方块受到足够冲击会被消灭/破坏；小猪也可能被滚落的方块压死（连锁反应是乐趣来源，务必保证物理传导正确）。

### 4.6 计分与星级

- 每只小猪、每个被破坏的方块给固定分数。
- **剩余未使用的小鸟** 每只给额外奖励分（鼓励高效通关）。
- 根据总分对照阈值给出 1~3 星评价，结算界面展示。

## 5. 系统与交互细节

### 5.1 游戏状态机

实现清晰的状态机：`LOADING → MENU → PLAYING → PAUSED → LEVEL_COMPLETE / GAME_OVER`。状态切换集中管理，避免逻辑散落。

### 5.2 摄像机 / 视口

- 关卡宽度通常大于屏幕，需要 **摄像机跟随**：发射后镜头跟随飞行的小鸟，命中或静止后平滑回到弹弓。
- 支持在发射前拖拽/查看关卡全貌（可选）。
- 实现世界坐标 ↔ 屏幕坐标转换函数，渲染层统一使用。

### 5.3 输入处理（鼠标 + 触摸统一）

- 统一封装 `pointerdown / pointermove / pointerup`，同时兼容鼠标和触摸，避免在移动端失效。
- 阻止触摸时的页面滚动 / 缩放（`touch-action: none`，`preventDefault`）。

### 5.4 音效与音乐

- 关键音效：拖拽皮筋、发射、碰撞、玻璃碎、小猪被消灭、爆炸、通关。
- 注意浏览器自动播放策略：**音频必须在首次用户交互后初始化 / resume**（AudioContext.resume）。

### 5.5 UI / HUD

- 顶部显示当前分数、关卡号。
- 显示剩余小鸟队列。
- 暂停 / 重玩 / 返回菜单按钮。
- 结算弹窗（星级 + 重玩 / 下一关）。

## 6. 需要特别注意的点（易踩坑）

1. **GitHub Pages 路径问题**：Pages 部署在 `https://用户名.github.io/仓库名/` 子路径下，所有资源引用务必使用**相对路径**（如 `./src/main.js`），不要用以 `/` 开头的绝对路径，否则资源 404。如用 Vite 打包，必须设置 `base: '/仓库名/'`。

2. **添加 `.nojekyll` 文件**：GitHub Pages 默认会用 Jekyll 处理，可能忽略下划线开头的文件夹。放一个空的 `.nojekyll` 在根目录可禁用此行为。

3. **物理稳定性**：
   - 设置合理的固定时间步长（fixed timestep），不要把 `requestAnimationFrame` 的可变 delta 直接喂给物理引擎，否则不同帧率下手感不一致、可能穿模。建议物理用固定 16.67ms 步进，渲染插值。
   - 高速小鸟可能穿透薄方块（隧穿问题）：开启连续碰撞检测，或避免方块过薄、限制最大速度。

4. **性能**：
   - 静止判定：方块/小鸟长时间静止后让其 `sleep`，减少计算。
   - 销毁离屏物体：飞出世界边界的物体要从物理世界移除，防止内存与计算泄漏。
   - 限制粒子特效数量。

5. **缩放与自适应**：游戏世界用固定逻辑分辨率，Canvas 按窗口大小等比缩放（letterbox），并处理 `devicePixelRatio` 保证高清屏不模糊。

6. **重置与重玩**：重玩关卡时必须**彻底清理**旧的物理世界、事件监听、定时器，否则会出现幽灵物体、事件重复绑定、内存泄漏。

7. **碰撞事件去重**：Matter.js 的碰撞事件一帧内可能多次触发，伤害结算需做防抖/去重，避免一次撞击扣多次血。

8. **版权**："Angry Birds" 是 Rovio 的注册商标与受版权保护的美术形象。**自学/开源项目请勿直接使用官方角色美术、名称作为商业用途**。建议使用原创的"鸟形/猪形"几何造型，README 中注明致敬性质，避免使用官方 logo。

9. **移动端体验**：触摸目标足够大，弹弓拖拽手感单独调参，禁用长按选中文本与右键菜单。

## 7. 开发里程碑（建议实现顺序）

Agent 应分阶段实现，每阶段产出可运行版本：

1. **M1 基础物理沙盒**：Canvas + Matter.js 跑通，地面 + 可掉落方块。
2. **M2 弹弓发射**：拖拽发射一只红鸟 + 轨迹预测 + 摄像机跟随。
3. **M3 目标与破坏**：加入小猪、材质方块、碰撞伤害、胜负判定。
4. **M4 关卡与计分**：数据驱动关卡、计分、星级、关卡切换。
5. **M5 多种小鸟技能 + 音效 + UI 打磨**。
6. **M6 移动端适配 + 性能优化 + 部署到 GitHub Pages**。

## 8. 部署流程（GitHub Pages）

1. 将代码推送到 GitHub 仓库（如 `main` 分支）。
2. 若为零构建项目：仓库 Settings → Pages → Source 选择 `main` 分支 `/ (root)`，保存即可。
3. 若用 Vite 构建：配置 `base`，用 GitHub Actions 自动 `npm run build` 并将 `dist/` 部署到 `gh-pages` 分支（可用官方 `actions/deploy-pages`）。
4. 在 README 顶部放置在线 Demo 链接、操作说明、截图/GIF。
5. 确认 `.nojekyll` 存在，所有路径为相对路径。

## 9. 验收标准

- 在桌面 Chrome / Firefox 与移动端 Safari / Chrome 上均可正常加载与游玩，无控制台报错。
- 弹弓发射手感顺滑，轨迹预测准确。
- 至少 3 个可玩关卡，胜负判定与计分正确。
- 重玩、切关后无残留物体或内存泄漏。
- 成功部署到 GitHub Pages 并可通过公开链接访问。
